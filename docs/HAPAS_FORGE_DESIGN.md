# Hapa's Forge: System Design & Mechanics

## 1. Core Philosophy: The Triad of Manifestation
Hapa's Forge is not just a form wizard; it is a **Ritual of Synthesis**. It treats the creation of new software agents (Avatars) as an alchemical process combining three fundamental energies.

### The Triad Framework
Every forged entity is composed of three prioritized stacks of existing Cards (Memories):

1.  **<span style="color: #ff4d4d">LOVE (Red) - Desire & Purpose</span>**
    *   **Concept:** The "Soul" or "Why".
    *   **Input:** Cards representing Goals, Mission Statements, Objectives, "What is wanted."
    *   **Function:** Defines the motivation and the target state the entity strives for.
    *   **Game Mechanic:** Acts as the primary filter for decision making.

2.  **<span style="color: #4d94ff">TRUTH (Blue) - Memory & Motion</span>**
    *   **Concept:** The "Mind" or "What is Known".
    *   **Input:** Cards representing Facts, Data, History, Documentation, Protocols.
    *   **Function:** Provides the context and constraints. It evaluates the feasibility of the "Desire."
    *   **Game Mechanic:** Acts as the knowledge base and library of facts.

3.  **<span style="color: #50c878">CONVICTION (Green) - Convergence & Will</span>**
    *   **Concept:** The "Hand" or "How".
    *   **Input:** Cards representing Backlogs, Tasks, Logic Flows, Tools, Execution Plans.
    *   **Function:** The energy that forces alignment between Love and Truth. It is the backlog of actions required to make the Desire a Reality.
    *   **Game Mechanic:** Determines the entity's agency, task prioritization, and "Move Set."

## 2. The User Experience (The Flow)

### Phase 1: The Altar of Synthesis (Input UI)
The screen is divided into two main zones: The **Inventory** (Card Library) and **The Forge** (The Triad slots).

**Visual Layout:**
*   **Left Panel:** A searchable, filterable list of the User's existing Cards (The "Ingredients").
*   **Center Stage:** Three vertical, droppable columns (The "Crucibles"), visualized as pillars or slots.
    *   **Left Pillar (Red):** "Stack Intentions Here"
    *   **Center Pillar (Green):** "Stack Execution Here" (Visualized centrally as the bridge)
    *   **Right Pillar (Blue):** "Stack Context Here"
*   **Interaction:**
    *   Users drag cards from the library into specific pillars.
    *   **Prioritization:** Users re-order cards within a pillar. Top card = Primary Driver. Bottom card = Subconscious influence.
    *   **Visual Feedback:** As cards are added, the pillars glow with their respective color intensity.

### Phase 2: The Incantation (Generation)
*   **Action:** User clicks a prominent central button: **"FORGE AVATAR"**.
*   **System Action:**
    *   Aggregates content from all stacked cards.
    *   Constructs a "System Prompt" utilizing the Triad Framework.
    *   Calls the LLM to generate the "Soul" of the new card.
*   **Animation:** Particles from the Red, Blue, and Green stacks stream toward the center, swirling and converging into a bright white light (The Singularity), revealing the new Card Face.

### Phase 3: The Character Sheet (Output UI)
The newly forged card opens in a "Character Sheet" view, reminiscent of D&D or RPG stat screens.

**Key Sections:**
1.  **Identity Header:**
    *   **Name:** (Generated by LLM, editable).
    *   **Class/Archetype:** (e.g., "Truth Seeker," "Code Architect").
    *   **Level:** Starts at 1 (Evolution count).
2.  **Visual Representation (Avatar):**
    *   **Image:** Initially a placeholder or generated pattern.
    *   **Image Generator Form:** The LLM pre-fills a prompt field describing the avatar based on its constituent cards.
    *   **Action:** User clicks "Manifest Appearance" (calls Image Gen API). The result populates the image slot directly.
3.  **The Triad Stats:**
    *   Visual bars showing the balance of Love/Truth/Conviction based on input volume.
4.  **Lore & Voice:**
    *   **Backstory:** A cohesive narrative synthesizing the input cards.
    *   **Voice Lines:** 10 sample phrases showing *how* this agent speaks/thinks.
5.  **Move Set (Capabilities):**
    *   List of "Skills" derived from the Green/Conviction stack.

## 3. Advanced Mechanics

### Mechanic A: Cultivation (Evolution)
A Forged Card is not static. It is a living Hypercore.

*   **The Loop:**
    1.  User opens an existing Avatar Card.
    2.  User drags *new* cards into the Triad Stacks (e.g., new knowledge, a changed goal).
    3.  User clicks **"CULTIVATE"**.
*   **The Logic:**
    *   The system feeds the *History* of the card + the *New Inputs* to the LLM.
    *   The LLM updates the profile: "Acknowledging that [Old Goal] is complete, and [New Data] has been acquired..."
*   **Rewards:**
    *   **XP Gain:** The Avatar gains XP. Level increases.
    *   **Visuals:** The border of the card evolves (Common -> Uncommon visuals).
    *   **Metadata:** The `evolutionCount` increments in the Hypercore.

### Mechanic B: The Phamiliar Bond (Pet Attachment)
This is the "Pokemon Trainer" mechanic.

*   **Concept:**
    *   **Pet Card:** A specialized tool/function (e.g., "Leo" - The Searcher).
    *   **Avatar Card:** The decision maker (e.g., "Project Manager").
*   **The Action:**
    *   Drag a **Pet Card** onto a specific "Phamiliar Socket" on the Avatar's Character Sheet.
*   **The Result:**
    *   **Inheritance:** The Avatar gains access to the Pet's "Move Set" (API tools).
    *   **Contextualization:** The Avatar wraps the Pet's raw function in its own "Voice" and "Love/Desire".
    *   **Example:** "Leo" knows how to `fetch(url)`. "Project Manager" Avatar *uses* "Leo" to `fetch(jira_ticket)` because it *Desires* (Red) to complete the backlog.

### Mechanic C: Public Discovery
*   **Registry:** Forged cards have a toggle: **"Register to Hapa Network"**.
*   **Function:** Publishes the Card's Hypercore Discovery Key to the public DHT.
*   **Interaction:** Other users can "Clone" the Avatar (adding it to their Blue/Truth stack) or "Hire" it (interacting via P2P).

## 4. Technical Architecture

### Data Structure (The Hypercore)
The Forged Card is a standard Card, but with a specialized `cardRecord` schema:

```typescript
interface ForgedCardRecord {
  type: 'avatar';
  level: number;
  experience: number;
  
  // The Triad Composition (Lineage)
  composition: {
    love: string[];       // Array of Card IDs
    truth: string[];      // Array of Card IDs
    conviction: string[]; // Array of Card IDs
  };

  // The Soul (LLM Output)
  profile: {
    archetype: string;
    bio: string;
    visualPrompt: string;
    voiceSamples: string[];
    moveSet: string[]; // Derived skills
  };

  // Connected Entities
  phamiliars: string[]; // Array of Pet Card IDs attached
  
  // Evolution History
  evolutionLog: {
    timestamp: number;
    addedCards: string[];
    summary: string;
  }[];
}
```

### Prompt Engineering Strategy
The system prompt for the Forge must be highly structured:

1.  **Context Injection:** "You are the Soul Forge. You are creating a <Archetype>."
2.  **Triad Parsing:** "Analyze these inputs. The user DESIRES <Red Stack>. The user KNOWS <Blue Stack>. The user WILL EXECUTE via <Green Stack>."
3.  **Synthesis Instruction:** "Synthesize a personality that creates a narrative bridge between these three forces. If Desire contradicts Truth, the personality should feel 'Conflict'. If Conviction is weak, the personality should feel 'Dreamy'."
4.  **Output Formatting:** Strict JSON for the UI to parse (Bio, Prompt, Voice Lines).

## 5. Design Aesthetic: "Cyber-Mystic RPG"

*   **Colors:** Deep Void Black backgrounds. Neon Red/Blue/Green highlights.
*   **Typography:**
    *   Headers: `Cinzel` or similar "Fantasy" serif font.
    *   Data: `Roboto Mono` for the "Tech" overlay.
*   **UI Elements:**
    *   **Stacks:** Look like card slots in a TCG (e.g., Magic: The Gathering graveyard/library).
    *   **Buttons:** "Gem" style buttons that look like they can be pressed physically.
    *   **Progress Bars:** Not just solid colors, but "fluid" animations (like liquid mana).
