# Thor's Hamma Card Generation Design

## Current Problem

Cards generated by Thor's Hamma are coming back with minimal data:
- Just names
- Empty or sparse lore
- No skills
- No practical "how to use" information
- No connection to hand context visible in UI

## Current Prompt Output Structure
```json
{
  "set_data": { "name": "", "lore": "" },
  "cards": [{
    "name": "",
    "lore": "",
    "skills": [{"name": "", "description": "", "type": "Passive"}],
    "stats": {"level": 1, "type": "Item"}
  }]
}
```

---

## Proposed: Fleshed-Out Card Schema

### Card Data Model
```typescript
interface ThorCard {
  // Identity
  name: string;                    // "The Synthetic Bard"
  subtitle?: string;               // "AI Music Generator"
  tier: 0-5;                       // Rarity/quality tier
  type: 'Concept' | 'Tool' | 'Technique' | 'Entity' | 'Principle';
  
  // Narrative
  lore: string;                    // 2-3 sentences of flavor text
  origin: string;                  // Where this came from (URL title/domain)
  
  // Practical Value
  skills: Skill[];                 // 1-3 skills per card
  desires: string;                 // What this card "wants" (use case)
  truths: string[];                // 2-4 factual statements about it
  
  // Actionable
  howToUse: string;                // Practical application guide
  synergies?: string[];            // What it pairs well with (from hand context)
  
  // Provenance
  sourceUrl: string;
  extractedFrom: string;           // Leo's analysis category
  handContextUsed?: string[];      // Which hand cards influenced this
}

interface Skill {
  name: string;                    // "Harmonic Generation"
  description: string;             // "Create music from text prompts..."
  type: 'Passive' | 'Active' | 'Triggered';
  cooldown?: string;               // For game-like flavor
}
```

## UI Display Plan

### HandCardView (Quick Inspector)
Show:
- **Name** + Tier badge
- **Thumbnail**
- **Skills** (names only, expandable)
- **Desires** (italic, single sentence)
- **Truths** (bullet list, 2-3 items)

### Full Card View (Library Inspector)
Show everything, plus:
- **How To Use** section
- **Synergies** (if hand context was used)
- **Provenance** (source URL, which AI analyzed it)
- **Card Lineage** (parent set, sibling cards)

### Context Badge
When a card was influenced by hand context, show a small badge:
```
🔗 Influenced by: [Card Name], [Card Name]
```

## Improved Prompts

### Leo (Scout) - Enhanced
```
You are "Leo" 🐕, the Scout.

MISSION: Deeply analyze this URL to extract maximum value.

ANALYZE:
1. IDENTITY: What IS this? (Tool, Service, Knowledge Base, etc.)
2. AUDIENCE: Who benefits from this?
3. CORE VALUE: What's the ONE thing this does best?
4. LOOT EXTRACTION: What specific skills/concepts can we capture?
5. PRACTICAL APPLICATION: How would someone USE this?

OUTPUT (JSON):
{
  "title": "Page title or name",
  "summary": "2-3 sentence summary of what this is",
  "category": "Tool | Resource | Knowledge | Service | Platform",
  "core_value": "The single most important thing about this",
  "loot_potential": [
    {
      "concept": "Name of extractable concept",
      "why_valuable": "Why this matters",
      "use_case": "How to apply it"
    }
  ],
  "target_audience": "Who this is for",
  "related_domains": ["AI", "Music", "Creativity"]
}
```

### Thor (Forge) - Enhanced
```
You are "Thor" 🐈, the Forge Master.

You transform raw intelligence into powerful Cards for the user's arsenal.

TARGET INTEL (from Leo):
${leoContext}

USER'S CURRENT HAND:
${handContext}

FORGING RULES:
1. Each card must be ACTIONABLE - something the user can DO with it
2. Lore should be evocative but also hint at practical use
3. Skills should describe specific capabilities
4. If hand cards are relevant, note synergies
5. Truths are FACTS, not opinions
6. Desires are what the card "wants to do" - its purpose

CREATE:
- 1 SET CARD (represents the source as a whole)
- 3-5 LOOT CARDS (specific valuable concepts/tools from the source)

OUTPUT (JSON):
{
  "set_card": {
    "name": "Evocative set name",
    "subtitle": "What this source is",
    "lore": "2-3 sentences describing this portal/source",
    "type": "Portal",
    "tier": 2,
    "truths": ["Fact 1", "Fact 2"],
    "desires": "What exploring this source enables"
  },
  "cards": [
    {
      "name": "Card Name",
      "subtitle": "Brief descriptor",
      "type": "Tool | Technique | Concept | Entity",
      "tier": 1-4,
      "lore": "2-3 evocative sentences",
      "skills": [
        {
          "name": "Skill Name",
          "description": "What this skill does, practically",
          "type": "Active | Passive | Triggered"
        }
      ],
      "truths": ["Factual statement 1", "Factual statement 2"],
      "desires": "What this card wants to help you do",
      "howToUse": "Step-by-step or practical guidance",
      "synergies": ["Hand card name if relevant"],
      "extractedFrom": "Which part of the source this came from"
    }
  ]
}
```
