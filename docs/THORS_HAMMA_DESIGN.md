# Thor's Hamma - Feature Design Document

## 1. Overview
"Thor's Hamma" is a new "Channel" (Left Menu Item) in the Hapa AG interface. It serves as a specialized forge for creating `<url>` cards. Unlike standard cards, these act as portals (using `iframes`) to external web resources, enriched by AI-generated context, lore, and "child" cards that extract skills, knowledge, or tools from that URL.

**Core Philosophy:**
- **The Forge:** We are not just bookmarking; we are *forging* a connection to an external reality.
- **The Portal:** The card is a window.
- **The Scout:** Agents (Leo/Thor) go ahead, scout the URL, and bring back intelligence tailored to the User's current "Hand" (their active loadout of needs/desires).

## 2. User Interface (UI)

### 2.1 Navigation
- **Icon:** `bolt` (or similar high-energy/electric icon).
- **Label:** "Thor's Hamma".
- **Location:** Left Sidebar, below "Hapa's Forge".

### 2.2 The Anvil (Input View)
- **Visual Style:** High-tech, neon-lit terminal. Dark background, pulsing grid lines.
- **Input:** A central, prominent "Target URL" input field.
  - *Effect:* Glitch text effect on typing.
- **Action Button:** "STRIKE" (Submit).
  - *Effect:* Thunder sound/visual shake on click.

### 2.3 The Process (Status View)
- **Visuals:** A scrolling terminal log showing the agent's actions.
  - `[SYS] Targeting URL...`
  - `[NET] Uplink Established.`
  - `[CAM] Snapshot Sequence Initiated...`
  - `[LEO] Scouting Content...`
  - `[THOR] Forging Assets...`

### 2.4 The Armory (Result View)
- **Layout:** Split view or Grid view.
- **Primary Artifact (The Set Card):**
  - **Type:** `set-url` (New Card Type).
  - **Visual:** A "Portal" frame. Displays the Screenshot initially.
  - **Interaction:**
    - *Click:* Expands/Activates `iframe` mode.
    - *Drag:* Draggable like a normal card.
- **Secondary Artifacts (The Loot):**
  - A collection of standard cards generated from the URL's content.
  - Displayed alongside the Set Card.

## 3. Technical Pipeline

### 3.1 Input & Capture
1.  **User enters URL.**
2.  **Backend (Electron):**
    - Spawns an offscreen `BrowserWindow`.
    - Loads URL.
    - **Capture 1:** `capturePage()` -> High-res screenshot (saved as Card Image).
    - **Capture 2:** `document.documentElement.outerHTML` -> HTML Snapshot (saved to Hypercore as `scroll-{timestamp}.html`).
    - **Cleanup:** Destroy window.

### 3.2 Intelligence (The Agents)
1.  **Leo (The Scout):**
    - **Input:** HTML Snapshot.
    - **Prompt:** "Analyze this page. What is it? Who is it for? What skills/tools does it offer?"
    - **Output:** `UrlContext` (JSON).
2.  **Thor (The Smith):**
    - **Input:** `UrlContext` + `HandContext` (Cards currently in user's Hand) + `HapaContext` (Protocol/Lore).
    - **Prompt:** "Synthesize a 'Set' that integrates this URL into the User's arsenal. Compare the URL's offerings with the User's current 'Hand' needs. Create cards that bridge the gap."
    - **Output:** `SetData` + `CardData[]`.

### 3.3 Fabrication (Hypercore)
1.  **Create Set:**
    - Type: `set-url`.
    - Metadata: `url`, `iframe_mode: true`.
    - Image: The Screenshot.
2.  **Create Cards:**
    - Type: `card` (Standard).
    - Content: Generated by Thor.
    - Image: Generated via Imagen/Vertex (Hell Run style).

## 5. Principles Applied
- **Astros/Futuristic:** Terminology (Uplink, Scout, Forge). Visuals (Neon, Terminal).
- **Loot/RPG:** The URL isn't a link; it's a "Location" or "Dungeon" that drops "Loot" (Cards).
- **Context:** The agents don't just summarize; they *contextualize* based on what the user is currently holding (Hand).
